Blood Spatter Analysis System – Measurement-Based Analytical Model

A measurement-based analytical model for bloodstain pattern analysis in forensic science. The system extracts geometric features from blood droplets using classical image processing and estimates angle of impact via deterministic ellipse fitting—no machine learning or deep learning. Designed for interpretable, forensically defensible results.

Problem Statement

Blood spatter patterns contain valuable forensic information for crime-scene reconstruction. Manual interpretation is subjective, time-consuming, and error-prone. This project builds a computational analytical model that extracts geometric features from droplets and estimates angle of impact using ellipse fitting, providing reproducible and geometrically rigorous results from 2D images.

Literature Review

  • Classical BPA (Bloodstain Pattern Analysis) relies on elliptical geometry and the arcsin(width/length) relationship for angle estimation.
  • Existing tools range from manual protractor-based methods to software-assisted ellipse fitting; many lack automation or rigorous geometric foundations.
  • Recent ML-based approaches offer automation but lack interpretability and forensic defensibility required in legal contexts.

Research Gap / Innovation

A purely measurement-based, deterministic pipeline using OpenCV and classical computer vision. No ML or CNN—fully interpretable, forensically defensible, and suitable for peer-reviewed forensic standards where black-box models are inadmissible.

Proposed Solution / Methodology

The pipeline follows: image acquisition → preprocessing (grayscale conversion, thresholding, Gaussian blur for noise reduction) → Canny edge detection → contour detection → ellipse fitting using OpenCV minAreaRect → feature extraction (major axis, minor axis) → angle estimation via θ = arcsin(minor/major). Measurement-based modeling is chosen over ML for interpretability, forensic admissibility, and deterministic reproducibility.

Input

2D blood spatter photographs (JPG/PNG). Grayscale conversion, thresholding, and noise reduction applied before contour extraction.

Measurement-Based Pipeline

OpenCV: contour detection, ellipse fitting (minAreaRect), major/minor axis extraction, arcsin formula for angle estimation. No ML.

Mathematical Model

A blood droplet striking a surface at an angle produces an elliptical stain. The aspect ratio of minor to major axis relates directly to the impact angle. Assumptions: elliptical shape, minor_axis perpendicular to impact direction, major_axis parallel; valid for typical forensic impact angles.

Angle of Impact Estimation:

θ = arcsin(minor_axis / major_axis)

minor_axis = smaller dimension (width) of fitted ellipse; major_axis = larger dimension (length); θ ∈ [0°, 90°]. Ratio is clamped to [0, 1] for numerical stability.

Derivation from fluid dynamics: the elliptical footprint’s eccentricity encodes impact angle. At 90° (perpendicular impact) the stain is circular (ratio = 1); at grazing angles the ellipse elongates (ratio → 0).
